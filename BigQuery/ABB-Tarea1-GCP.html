Actividad: Análisis Integrado de Ventas y Comportamiento del Cliente en BigQuery

Objetivo:

Realizar un análisis de datos para identificar patrones y relaciones entre las ventas, el comportamiento del cliente y las interacciones con el soporte. Los estudiantes utilizarán BigQuery para procesar y analizar los datos proporcionados, generando insights que puedan mejorar la estrategia de ventas y servicio al cliente.

Instrucciones:

  1. Acceder a los Datos

    • Se te proporcionan tres conjuntos de datos almacenados en Google Cloud Storage. Los datos están comprimidos en formato .gz y puedes acceder a ellos utilizando los siguientes URIs:
      • Ventas Detalladas: tienda-analisis-comportamiento-2024/ventas_detalladas.csv.gz
      • Comportamiento del Cliente: tienda-analisis-comportamiento-2024/comportamiento_cliente.csv.gz
      • Soporte al Cliente: tienda-analisis-comportamiento-2024/soporte_cliente.csv.gz

  2. Crear Tablas en BigQuery

    • Paso 1: Accede a BigQuery a través de Google Cloud Console.
    • Paso 2: Crea un nuevo conjunto de datos en tu proyecto llamado analisis_comportamiento.
    • Paso 3: Importa cada uno de los archivos comprimidos como una nueva tabla en tu conjunto de datos. Los nombres de las tablas deben ser:
      • ventas_detalladas
      • comportamiento_cliente
      • soporte_cliente
    • Paso 4: Durante la importación, asegúrate de seleccionar “CSV” como formato del archivo y activa la opción “Autodetectar” para que BigQuery identifique automáticamente el esquema de cada tabla.

  3. Realizar Consultas SQL

    • Utiliza BigQuery para realizar las siguientes consultas y análisis sobre los datos:

    • Consulta 1: Identifica el método de pago más común por categoría de producto.

      SELECT categoria, metodo_pago, COUNT(*) AS frecuencia
FROM `tu_proyecto.analisis_comportamiento.ventas_detalladas`
GROUP BY categoria, metodo_pago
ORDER BY frecuencia DESC;

    • Consulta 2: Analiza la relación entre el carrito abandonado y las solicitudes de soporte.

      SELECT carrito_abandonado, COUNT(*) AS cantidad, AVG(soporte_solicitado) AS soporte_promedio
FROM `tu_proyecto.analisis_comportamiento.comportamiento_cliente`
GROUP BY carrito_abandonado;

    • Consulta 3: Evalúa la satisfacción del cliente según el tipo de contacto con el soporte.

      SELECT tipo_contacto, AVG(satisfaccion) AS satisfaccion_promedio
FROM `tu_proyecto.analisis_comportamiento.soporte_cliente`
GROUP BY tipo_contacto;

    • Consulta 4: Determina si existe una correlación entre la duración de la llamada y la satisfacción del cliente.

      SELECT duracion_llamada, AVG(satisfaccion) AS satisfaccion_promedio
FROM `tu_proyecto.analisis_comportamiento.soporte_cliente`
GROUP BY duracion_llamada
ORDER BY duracion_llamada;

  4. Preparar un Informe Final Incluye capturas de pantalla de cada consulta SQL junto con los resultados obtenidos.

  5. Entrega del Informe

    • El informe final debe ser preparado en formato PDF y entregado a través del aula virtual (Blackboard). Asegúrate de incluir los nombres de los integrantes de grupo, la fecha, y el título del informe.
